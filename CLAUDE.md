# 라인업 메이커 (가칭) - 프로토타입 기획 정리(첨부한 프로토타입 사진 참고할 것)

## 1. 프로젝트 개요

축구 라인업을 만들고 지인들과 공유하는 모바일 친화 웹앱.
기존 "LineUp 11"과 유사하지만 다음 두 가지가 핵심 차별점:

1. **동적 포메이션**: 4-4-2, 4-3-3 같은 고정 템플릿에 선수를 끼워 넣는 방식이 아니라,
   사용자가 모바일에서 터치로 선수를 필드 위 원하는 위치에 자유롭게 배치/이동(이건 "LineUp 11" 거의 동일).
2. **쿼터별 댓글**: (프로토타입 사진 첨부 참고) 라인업 공유로 끝나는 게 아니라, 쿼터(1~N쿼터)별로 지인들이
   댓글을 남길 수 있어 "공유 → 피드백" 루프가 생김.
3. UI 색은 파란색톤으로 개발(내가 Chelsea FC 팬이라서 개인적인 추구미)
   목적은 단순함 유지: 복잡한 기능을 늘리지 않고 "라인업 짜기 + 공유" 라는 핵심 동작에 집중.

## 2. 핵심 기능

### 2.1 선수 명단 관리

- 이름 + 번호로 선수 추가
- 선수 명단(스쿼드)에서 선수 삭제
- 명단은 팀 전체 기준으로 유지되고, 각 쿼터에서 "필드에 배치된 선수 / 대기(벤치) 선수"로 구분됨

### 2.2 포메이션 배치 (핵심 차별점)

- 정적 포메이션 슬롯 없음. 필드는 % 좌표 기반의 자유 캔버스
- 대기 명단에서 선수를 탭하면 필드 위 빈 자리에 자동 배치됨
- 필드에 배치된 선수는 손가락(터치) 또는 마우스로 드래그하여 자유롭게 위치 이동
- 배치된 선수를 다시 탭/X 버튼으로 제거하면 대기 명단으로 복귀
- 작성자를 제외한 다른 인원들은 라인업 및 포지션을 수정할 수 없음

### 2.3 쿼터 관리

- 쿼터를 원하는 만큼 추가/삭제 가능 (예: 4쿼터 축구 → 1~4쿼터)
- "빈 쿼터 추가": 새 쿼터를 배치 없이 생성
- "쿼터 복제": 현재 쿼터의 선수 배치를 그대로 복사해 새 쿼터 생성 (교체 인원만 수정하면 되도록)
- 각 쿼터는 독립적인 선수 배치 + 댓글 목록을 가짐

### 2.4 쿼터별 댓글

- 각 쿼터마다 별도의 댓글 스레드
- 이름(선택) + 텍스트로 댓글 작성
- (확장 고려) 로그인 없이 공유 링크만으로 댓글 작성 가능하게 하면 지인 참여 장벽이 낮아짐

### 2.5 공유

- 현재 라인업(전체 쿼터 포함)을 공유 가능한 형태로 내보내기
- 1차: 공유 링크 (문서 단위 URL)
- 2차 확장: 라인업을 이미지로 캡처해서 카카오톡 등으로 공유 (텍스트 링크보다 가독성 높음)

## 3. 데이터 모델 (제안)

```
Player {
  id: string
  name: string
  number: string
}

PlacedPlayer {
  playerId: string
  x: number   // 0~100, 필드 가로 위치(%)
  y: number   // 0~100, 필드 세로 위치(%)
}

Comment {
  name: string
  text: string
  createdAt: timestamp
}

Quarter {
  id: string
  label: string            // "1쿼터" 등
  players: PlacedPlayer[]
  comments: Comment[]
}

Lineup {
  id: string                // 공유 URL에 사용
  teamName: string
  squad: Player[]
  quarters: Quarter[]
  createdAt: timestamp
  updatedAt: timestamp
}
```

## 4. UI 구성

- 상단: 팀명(인라인 편집) + 공유 버튼
- 쿼터 탭 바: 가로 스크롤, 활성 쿼터 강조, + (빈 쿼터) / 복제 버튼
- 필드 영역: 세로형 축구장 (공격 방향 위쪽, GK 아래쪽), 배치된 선수는 번호 원형 토큰 + 이름 라벨
- 대기 명단: 가로 스크롤 칩 목록, 탭하면 필드에 배치
- 선수 추가 폼: 이름/번호 입력 + 추가 버튼
- 댓글 영역: 현재 쿼터 기준 댓글 목록 + 입력창

## 5. 기술 스택 (제안)

- Frontend: React (모바일 퍼스트, 터치 이벤트 기반 드래그)
- Backend/DB: Firebase Firestore (라인업 문서 단위 저장, 가벼운 JSON 구조로 충분)
- Hosting: Vercel
- 공유: Lineup 문서 ID 기반 URL, 추후 이미지 캡처 라이브러리(html-to-image 등) 검토

기존에 사용해본 React + Firebase + Vercel 조합 그대로 사용 가능. 별도 백엔드 서버 불필요.

## 6. MVP 개발 순서 제안

1. 선수 명단 CRUD + 필드 자유 배치 (드래그)
2. 쿼터 추가/삭제/복제
3. 쿼터별 댓글
4. 공유 링크 (Firestore 문서 저장 + 조회 페이지)
5. (확장) 비로그인 댓글, 라인업 이미지 내보내기, 출전 기록 누적
